淘宝网CSS研究【1】css渲染效率的探究

要求：以较小的牺牲，做到分层体验。让优秀的浏览器渲染的更快、更好。

本着这个要求和目的，这方面以前是没接触过的，而且把淘宝的CSS下载下来，到现在还没能够领悟到，只是粗浅的看看了，没理解到其中的精粹，好长好长好的的。

首先对于CSS的渲染效率，以前没有任何接触

结合淘宝的CSS和网上的相关资料，对于CSS的渲染进行一下总结和梳理。

1、不要使用过深的CSS路径，不要使用通用选择器，ID选择器>class选择器>tag选择器>通用符，选择器是从右到左读取的。
选择器的概念，我们可以知道下面这个选择器并不高效：#main-nav &gt; li { } / 看着很快实则很慢 */

思考下这东东：

以下为引用的内容：

#main-navigation li a { font-family: Georgia, Serif; }

你可能不需要从 a 选择器开始（如果你只是想换个字体）。下面这个可能更高效些：

以下为引用的内容：

#main-navigation { font-family: Georgia, Serif; }

如果你有一个选择器是以id作为关键选择符，请不要添加多余标签名上去。因为ID是唯一的，你不要为了一个不存在的理由而降低了匹配的效率。

  • 不赞成 - button#backButton { }
  • 不赞成 - .menu-left #newMenuIcon { }
  • 建议用 - #backButton { }
  • 建议用 - #newMenuIcon { }

2、滤镜的一些东西不要去用，滤镜是一个非常毫资源的东西;特别是一些羽化、阴影和一个前透明的效果并且还要写HACK;

IE的滤镜除了比较消耗资源外也有兼容性问题。当中有令PNG透明的滤镜，可采用GIF或JPG似透非透的办法来避免使用此滤镜。建议只在IE6应用GIF透明，因为IE7以上已经支持了PNG透明。

  • 不赞成，滥用IE滤镜因为消耗资源外也有兼容性问题。
  • 建议用，最好选择其它方法能避免使用滤镜。

3、一个页面上少用绝对定位
尽量少用、变通的原则。

4、background背景图片的平铺

一张宽高1px的背景图片，虽然文件体积非常之小，但渲染宽高500px的板块需要重复平铺2500次。提高背景图片渲染效率跟图片尺寸及体积有关，最大的图片文件体积保持约70KB。

  • 不赞成 - 宽高8px以下的平铺背景图片
  • 建议用 - 衡量适中体积及尺寸的背景图片，色彩少的图片要作成gif图片;

5、让属性尽可能多的去继承

尽可能让一些属性子可以继承父，而不是覆盖父;

7、能简写的一些就简写;

例如

#zishu{pading-top:10px;padding-right:50px; padding-left:50px;padding-bottom:4px;} 改成：#zishu{padding:10px50px4px50px}

这个对渲染速度没有影响;只是少几个字符;

8. display与visibility的差异

他们用于设置或检索是否显示对象。display隐藏对象不保留物理空间，visibility为隐藏对象保留占据的物理空间。当浏览器渲染被占据的物理空间时，会有所消耗资源。

  • 不赞成 - visibility:hidden;
  • 建议用 - display:none;

将DEDECMS后台进行优化精简、增强管理用户体验

DEDE作为国内顶尖的CMS，以可扩充灵活性和模板开发易用性为其亮点，但是用过的人都会觉得后台太过于复杂，甚至于臃肿，后台的载入慢，菜单项目及其多，这样对于我们开发人员尚且觉得比较麻烦，那么对于 企业的网站管理者 或者说 我们的客户来说，是很不方便的。后台的易用性也导致了很多人自己在位客户开发网站的时候为考虑后台管理者的方便性自己开发后台。

记得有一次，湘潭某网络公司说找人开发后台什么的，也就是开发他客户企业站的后台，顺便加了一下QQ了解一下，然后给他建议直接用DEDE CMS，然后对方提出DEDECMS的后台过于复杂，不需要这么“专业”的后台。

其实嘛，CMS只是提供一个通用的功能，让大家提高开发效率，不怕不方便，不怕功能过于多，怕的是自己不会修改，不会精简，不懂得如何去 “去其糟糠，取其精华”。那么我本着以后通用、后台管理方便的原则，将DEDE的后台精简，这个精简的后台界面提供给客户使用，提高友好性。而原复杂后台当然也没删掉，只是隐藏起来，供网站的维护者以后扩充使用。

做一件事情，我们不只需要提高精度，还需要提高做事的效率。

javascript 选择 class 类

下面写一个遍历某类的封装函数。

function (element,class){
    var elementS = document.getElementsByTagName(element);
    for (var i=0; i<elementS.length; i++)
    {
        if(elementS[i].className == class)
      //执行的操作
       return false;
    }
}

用javascript练习实现的栏目高亮

前阵子在做网站的时候想实现【网站当前栏目高亮显示】，用 服务端语言实现的话也简单，对当前栏目的 变量进行附加判定加一个class高亮类，但是了，我用的那个程序是网上套用的开源程序，那么我找了好久也找不到有那个 当前栏目的变量部分，所以一时间搁浅了。
最近在看【JavaScript DOM编程艺术】里面正好有实例讲到了这种 【当前页面标志】，当然这上面是简单的例子，有不足的地方，我在这里把他补充完整并且考虑完善。
做了两点改进：
1、函数参数化，便于重用通用。
2、给当前网址和文章栏目网址同时加上一个 结束符，然后进行匹配，这样减少了 诸如 1 匹配 123成功 的问题，列如 当前栏目是 http://127.0.0.1/website/category.php?cid=31， 那么 http://127.0.0.1/website/category.php?cid=3的栏目也会高亮，那么我的解决方式就是加上一个 结束符。
Javascript：

<script type=“text/javascript”>// 
<![CDATA[
    window.onload = function(){
    function highlightpage(hightlightID,curclass){
    var nav = document.getElementById(hightlightID);
    var links = nav.getElementsByTagName(“a”);
    var j = 0;
    for (var i=0; i        <links.length; i++)
        {
            var linkurl = links[i].getAttribute(“href”) + “/“;
            var currenturl = document.location.href +“/“;
            if(currenturl.indexOf(linkurl) != -1)
        {
            links[i].parentNode.className = curclass;
            j++;
        }
    }
    if(j > 1)
    {
        links[0].parentNode.className = null;
    }
    }
        highlightpage(“nav”,“cur”);
    }
// ]]>
</script>

HTML结构：

<div id=“nav”>
<ul>
    <li><a title=“红灯街首页” href=“http://127.0.0.1/hongdengjie/“>热门的</a></li>
    <li><a title=“最新的” href=“category.php?cid=3”>最新的</a></li>
    <li><a title=“最佳的” href=“category.php?cid=4”>最佳的</a></li>
    <li><a title=“热门的” href=“http://127.0.0.1/hongdengjie/post.php“>发表文章</a></li>
</ul>
</div>

当然这样前端实现的栏目高亮还有至少一个不足，文章内页的栏目可能无法高亮，因为URL规则不同，当然具体情形具体对待解决嘛

今天第一次独自走出校园实地谈业务及其需求

        昨天从同学那的得知一个单，也就是网站设计制作。有一段时间没有接单了，所以了手也有点痒，并且还想着攒钱买台新本本，所以了，马上就联系了。在QQ上和他（一个我们学校的研究生在离学校不远的公司上班）聊需求进行网站制作，他最开始发帖的价格写的很低，从我的理解上讲是有很多人希望花最少的价格做好事情并且对市场的规律和劳动力的多少不了解，当然他这个价格也肯定是找不到人帮做这个小单的。但是本着开拓市场试试看的原则那么我也就和他洽谈了，网络上谈业务还是不够好的，比如价格啊什么的还是说不清楚的，那么在聊的过程中我也就基本避价格不谈，然后见面的时候详细说明，这样也显示的出诚意，但是了在我们聊天的过程中，因为是一段一段的打，其实他有一句的意思是 如果觉得那个价格 可以 那么就去他公司再谈，而我以为他的意思是 如果我们有能力搞定这个单，那么就去他公司。当然网上是能不能够搞定，具体的话肯定也要见面才能确定的，毕竟如果不是很熟悉的，要办相关手续。

        那么了，今天我就好好的准备了，然后去了他公司（湘潭某电子公司），感觉很热心的一个朋友，公司在离我校不远的另外一个学校里面，我走进去还觉得比较规范，然后了，就坐在他的办公的地方，并且使用了电脑，和他仔细谈需求，资料什么的，基本搞定，做好了需求分析，自己心中也对这个单的难度，工作量进行了评估，评估也是很合理的，毕竟是自己第一次走出校园开拓市场，对于自己的劳动价值，也极大的压缩了劳动价值体现，报了一个最低的数字，也是自正式接单寻单以来的第一次降低姿态的数字。然后他说 他昨天已经跟经理说了昨天那个价格（纯白菜价），然后说 要跟经理说下。 然后送我到门口，在这过程中我也给他讲了下 网络公司的 价格，整个湘潭的 市场等的内容，当然如果我们接这个 也是以高质量 而且 算是很低的价格来完成，毕竟好的作品，好的服务，才有好的口碑，更是长久发展之道。

        然后回来以后，他又Q我。一开始我以为是讲价，这个价格已经是最低的了，也不可能还价的了。后来说 是 因为 昨天答应了那个白菜价什么的，都跟经理说了，经理对他有意见了。天哪！！我没有印象我答应了哦，但是经过仔细查看以后确实有异议的地方，那就怪我理解不对吧，毕竟昨天的聊天是一段一段出来的，中间有间隔开来了，他如果把责任推到我身上（我觉得大部分也是我的责任）经理也会觉得他从中拿了回扣，或者和我和他串通的。我也很过意不去的，给别人带来了麻烦，一般好多领导疑心是很重的，当下属的不容易哦。当然这个单我也不可能降低成白菜来弄，这也是一个原则。当然如果这个搞定下来的话，会认认真真保质保量的完成的，因为这其中我也有过失，给他人造成了不必要的麻烦。当然，很多东西要注重，长久的发展才是好的，有希望有前景有未来。

        记得以前第一次谈需求的时候，被那个老师宰的，完全是免费的了，然后那种单也没有进行长久的维护了。后来有幸和学长合作获得了各方面的很多经验，自己也在校内谈了几次需求，出了那个第一次谈的，其他的还是相对合理的，虽然有吃亏但是也没吃太多亏，依稀记得曾经第一次的时候当时还要忙于期末考试那时候累死累活的尽心尽力的，当然的出来的结果也是很另自己另别人满意的。然后走入稍微正规以后也懂得了慢慢的用心去维护，用心去搞售后服务，毕竟在运行过程中或者出现一些小错误，或者他们不会操作什么的，自己都是很用心的帮着弄的，毕竟如果负责网站工作的老师或者是工作人员把工作搞砸了而我又不帮忙的话也是对他人的一种不负责任，更是有损自己的口碑和形象，当然这也只是一点小事，自己能尽力则尽力，自己的一点小事，如果没有帮到别人，那么别人可能就是一件大事了。然后还记得在学长的指导下跟其中一位老师讨价还价。在这些个过程中也碰到了一些问题，比如老师喜欢把网站一屏展示完，这样很短显得内容不充实等，当然我也只能够应用自己所了解的知识进行分析讲说利弊，当然最后决策权还是在客户手中，毕竟是为客户服务。

        当然这么久以来，学会了一些道理。比如 用心做事，售后服务，口碑，形象等，一个人或者说一个团体要发展就要遵守一些守则，要用心，貌似这些东西古代的道德书籍什么的里面讲的多。
