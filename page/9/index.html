最精简全兼容CSS代码实现1像素圆角

看到了莫有，1像素的圆角，代码精简哦，兼容主流浏览器，当然比不上CSS3的border-radius精简哦。



今天在群里看到一个分享代码写的1px圆角，然后用那代码写了下，精简是精简，但是不兼容。于是稍微修改完善了一下改兼容了。 以前还从没有用过 margin: -1px 0; 因为写过这样的不兼容，解决方法还是有的，下面就是代码了。

.radius,.radius_inner{border-width:1px; border-color:red;}
.radius {display: inline-block; border-style:none solid;}
.radius_inner {border-style:solid none; float:left; margin: -1px 0; position: relative; *width:100%;}

<div class=“radius”>
<div class=“radius_inner”>看到了莫有，1像素的圆角，代码精简哦，兼容主流浏览器，当然比不上CSS3的border-radius精简哦。</div>
</div>
以前一般写兼容的圆角用图片滑动门使用较多，有个时候用下N个标签嵌套，1像素的圆角也是最近工作的时候才碰到，刚开始看着好纠结的，这么一点点设计师为啥么不直接用直角啊，搞的我刚开始觉得是出图不严谨导致的误差效果，因为以前确实发过来的设计图有个时候有的地方不是很严谨。好吧，下次有1px的圆角又要求兼容IE系列的话就直接用这个代码丢进去了。

css分割线的几种写法--源自新浪微博的代码的思考

前几天看到细心的 【@点头猪】 大神竟然在逛新浪微博发现里面有个类似分隔线的友情提示，并对新郎微博的写法提出了一点想法，我自己也捣鼓着写了一下，下面是自己思考后得出的几个方法！

方法一** 单标签奇葩实现：**

.demo1{padding:0 30px 0;
       margin:7px 0;
       line-height:1px;
       border-left:100px solid #ccc; /*长度的刚性局限性*/
       border-right:100px solid #ccc; /*长度的刚性局限性*/
}

<div class="demo1">5分钟前，你看到这分钟</div>

代码很简洁！如果考虑文字会多，加个不换行和截断吧！

方法二** 巧用背景色层叠实现：**

.demo2{height:1px; border-top:1px solid #ccc; text-align:center;}
.demo2 span{    position: relative;
        top:-7px;
        margin-top:7px;
        background:#F60; /*颜色局限性*/
        padding:0 10%
       }

<div class="demo2"><span>5分钟前，你看到这分钟</span></div>

代码相对简洁！自适应宽度！

方法三 浮动：

.demo3_b{background:#ccc; margin-top:8px; float:left; width:20%; height:1px;_overflow:hidden}
.demo3_t{padding:0 14%; width:32%; float:left;}

<div class="demo3_b"></div>
<div class="demo3_t">5分钟前，你看到这分钟</div>
<div class="demo3_b"></div>

新郎微博使用该方式，代码很多，文字多的话换行的线也不居中,还需要闭合清楚浮动。

方法四 inline-block：

.demo4_b{background:#ccc; margin-top:8px; display:inline-block; width:20%; height:1px; _overflow:hidden;  vertical-align: middle;}
.demo4_t{display:inline-block; padding:0 14%; width:30%; vertical-align: middle;}

<span class="demo4_b"></span>
<span class="demo4_t">5分钟前，你看到这分钟,是不是很难看啊</span>
<span class="demo4_b"></span>   <!-- 中线也居中鸟···，多行真的很难看啊 -->

文字可换行

 

看到微博我就想啊想，第二天给 @大前端 看了一眼，请教他有什么看法，然后他稍微讲了下用border不就可以了。然后了，我就开始动手了，写了一个 代码 两个标签，外层写边框和padding里面用 position: relative; top:-x px 或者margin-top。然后 @ 大前端 一看，还说还可以优化的呀，就一个标签嘛，然后想啊想，想出来了，最后，经过 在微薄上和 @小秦同学 各种互相质疑代码，然后了然后结合各种。。。。。。

css布局之浮动、定位、负边距三者之间的灵活结合利用

在我们进行页面布局的时候，有时候需要利用浮动、定位、负边距各自的属性特定灵活的取长补短从而达到我们的目的。来看几个示例吧··。

四、 浮动与相对定位孕育的神奇的未知宽度水平居中


大多数时候我们用惯了 width:xxx px + margin:0 auto,而如何实现未知宽度水平居中了。这里有个神奇的方案。

结构如下：

<div class=“wrap”>
<div class=“member_paging_wrap”>
<div class=“member_paging”>

<a href=“#”>1</a><a href=“#”>2</a><a href=“#”>3</a><span>…</span><a href=“#”>4</a><a href=“#”>5</a><a href=“#”>6</a>

</div>
</div>
<div class=“member_paging_wrap”></div>
</div>
实现本方案的核心代码：

.member_paging_wrap{float:left; position:relative; left:50%; }

.member_paging{ float:left; position:relative; right:50%; }

分页的外套父容器.member_paging_wrap 通过float:left获得的其子容器 member_paging加上的member_paging作为宽度为186px。子容器.member_paging 是靠.member_paging{ float:left; }获得宽度的。(这里利用的是浮动的包裹性，浮动元素里有内容未本身没定宽度将识别子容器宽度并获得其宽度)。

然后通过相对定位 position:relative; left:50%，则.member_paging_wrap距离左边的50%是其外容器.wrap/2的值为480px。(position:relative **的百分比是按其父容器的宽度来确定的）**。

这样一来member_paging_wrap的左边缘在整个容器.wrap的中线上来。当然这还不是我们想要的。

我们要将里面的分页.member_paging 往左拉其分页包裹外容器的一半距离才能实现居中。

那么继续

.member_paging{ float:left; position:relative; right:50%; }

同样的原理，居中了。

参考自：http://matthewjamestaylor.com/blog/beautiful-css-centered-menus-no-hacks-full-cross-browser-support

css每个属性都有自己的特点,上面几个简单的例子灵活的运用了浮动、定位、负边距各自的特点，让他们“暧昧”。灵活运用属性各自的特点能够化繁为简、化腐朽为神奇之奇效。

.member_paging{ float:left; position:relative; right:50%; }

分页的外套父容器.member_paging_wrap 通过float:left获得的其子容器 member_paging加上的member_paging作为宽度为186px。子容器.member_paging 是靠.member_paging{ float:left; }获得宽度的。(这里利用的是浮动的包裹性，浮动元素里有内容未本身没定宽度将识别子容器宽度并获得其宽度)。

然后通过相对定位 position:relative; left:50%，则.member_paging_wrap距离左边的50%是其外容器.wrap/2的值为480px。(position:relative **的百分比是按其父容器的宽度来确定的）**。

这样一来member_paging_wrap的左边缘在整个容器.wrap的中线上来。当然这还不是我们想要的。

我们要将里面的分页.member_paging 往左拉其分页包裹外容器的一半距离才能实现居中。

那么继续

.member_paging{ float:left; position:relative; right:50%; }

同样的原理，居中了。

我个人倾向于使用 zoom:1触发ie6 haslayout ，因为这个属性是缩放属性，值为1就是按原大小缩放，所以对页面没任何可见影响。安全绿色！

一、 使用br标签的clear属性清除浮动

<div class=“father”>
   <div class=“”float_left””></div>
   <div class=“”foat_right””></div>
</div>

 

将
放在浮动元素之后而清除浮动。

该方式对IE无效,如果父容器没设置过width height position(relative absolute) zoom属性。

再触发一下haslayout

.father{zoom:1}

这种方式增加了一个额外的标签而且还要配合zoom:1才能达到效果,在父容器设置过width height position(relative absolute) 属性的时候不需要配合zoom:1。

二、增加一个div使用css的clear属性清除浮动

<p align=“left”>.clear {clear:both}</p>
   <div class=“father”>
   <div class=“”float_left””></div>
   <div class=“”foat_right””></div>
</div>

 

将


放在浮动元素之后而清除浮动。

该方式对IE无效,如果父容器没设置过width height position(relative absolute) zoom属性。

再触发一下haslayout

.father{_zoom:1}

这种方法类似br的方式但优于br的方式，因为在页面重构过程中基本要求XHTML和CSS样式相分离，增加了一个额外的标签。

 

一般情况下，父容器的width值是会设定的，所以一般情况下，使用上述两种方式不需要使用zoom:1从而这用在浮动元素后加入清除浮动的元素就行了，使用也较为方便，但是增加了一个额外的没有用的元素。

三、使用overflow .cd方法闭合浮动

使overflow属性来清除浮动有一点需要注意，overflow属性共有三个属性值：hidden,auto,visible。可以使用hiddent和auto值来清除浮动，但不要使用visible值(visible只对IE浏览器有效)。基于IE 6 的haslayout 问题，我们定义一个单独的class应用于父元素而闭合浮动(使用auto可能会产生滚动条)。

.cd{overflow:hidden; _zoom:1}

 

将该class用于内部有浮动的父元素就行了。

 

优点：简单有效，代码简短，没有增加额外标签，便于公用.

缺点：有截断，如果父元素内部有绝对定位或者其他超出父元素的元素时，并且会截断CSS3的一些效果，有点暴力！

 

四、 使用伪类clearfix方法闭合浮动

首先介绍一下:after和:before伪类。在偏向标准的浏览器中，可以通过:after 和:before在标签元素的后面和前面显示更多的信息。，:after和:before相当于在元素前面和后面插入两个行内元素块。

因为:after和:before是行内的所以使用display:block块化，行内元素是不可以清除浮动的，visibility:hidden将元素隐藏但占据位置，height:0将元素高度设为零使不影响布局,clear:both闭合浮动，content 必须填写不为空的任意值，:after :before所“生成”的元素才存在,zoom:1 触发IE haslayout。

.clearfix:before,
.clearfix:after {
content: “.”;
display: block;
height: 0;
visibility: hidden;
}
.clearfix:after {clear: both;}
.clearfix {zoom: 1;}

其实只使用clearfix:after就可以达到清除浮动的效果，但只使用clearfix:after时在跨浏览器兼容问题会存在一个垂直边距叠加的bug。
优点：没有增加额外的标签，便于公用。
缺点：相比.cd方法代码较长
