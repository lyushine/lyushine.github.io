一个很好的hasLayout解释

很多时候，CSS在IE下的解析十分奇怪，明明其他浏览器好好的，而在IE下却出了问题，并且问题十分诡异—-例如一个比较典型的BUG就是设置border的时候，有时候border会分开，刷新或者滚动条的时候，又会自动连接起来（这个我碰到过）。 那么我认为碰到这样的诡异现象的时候第一印象考虑出发hasLayout。

hasLayout到底是什么了？ hasLayout是IE浏览器专有的一个属性，用于CSS的解析引擎。有时候在IE下复杂的CSS设置解析起来会出现BUG，其原因可能与hasLayout没有被自动触发有关，我们通过一些技巧，手动触发hasLayout属性就可以解决Bug了。这是针对IE的偏方。

hasLayout的触发方法有很多种，例如 width height 值，设置postion 为relative等。 width heigh postion 都会带来一定的副作用。一个很好的流行的解决方法是 “zoom:1” 。在极少数情况下也是会失效的例如DHTML的时候。 这时借助 “position:relative”来触发尽管会带来副作用。 总之,“zoom”是最常用、最安全、成本最小的触发hasLayout的方式，一般情况下使用该方式。

热烈祝贺湖南科技大学校卫队网站成功上线【樱缘网络视觉工作室】

首先，我个人认为作为一个学生组织能够有一个属于自己的网站，这样有利于部门的宣传，有利于部门工作的展现和与广大师生互动做好本职工作。而作为一个学生组织在保卫处领导的要求下自己处理网站制作事宜也不容易。在我们帮助校卫队制作网站的过程中，也充满了一些经历。比如 约见处领导的时间冲突，要等领导很久。在这过程中校卫队队长有个时候也因为一点小事就被领导批评，在领导下面，网站的栏目，文章等都需要认真审核，不允许有一点差错。并且我们在提交给处领导的时候，处领导也会提出许多改进，修改也很麻烦，无论是界面要求还是程序都做了很多修改，这一点确实很繁琐。校卫队长在协调工作中尽心尽责。当然很高兴今天湖南科技大学校卫队的网站终于上线了。

查看演示：http://dep.hnust.cn/xwd/index.aspx

校卫队网站 以突出保卫安全的风格体现主题。前端 页面 主要继续采用html+css布局，并以jquery为主体的javascript下加入了多个动态元素，提高用户体验。后端程序采用 asp.net 书写。

这个网站，也是【樱缘网络工作室】 自正式组建以来，第一次以校园团队组员多人进行制作，其中更得到了宝贵的分工合作经验。

php数组输出的方法【特别适用于调试】

PHP编程过程中常常需要对查询出来的数据进行检测和调试，特别是对从表中查询出来的数据在一个数组中，那么现在以下数据的三种输出方法：

<?php
$bbbb=array("11"=&gt;"aaa","22"=&gt;"bbb");
//只能输出值value不能输出key
foreach($bbbb as $color)
echo $color;
//value与key都可输出
foreach($bbbb as $key=&gt;$value)
echo $key."=&gt;".$value;
//value与key都可输出
while($color=each($bbbb)){
echo $color['key'];
}
?>

CSS最小高度，容器下有内容则自动撑开，无内容则保留最小高度 兼容各主流浏览器

最近的一个网站因为本身提供的文章内容不足，用到了CSS最小高度，顺便我理清一下思路，复习一下这个小知识点。

实现效果——兼容IE6\IE7\FF 容器下有内容则自动撑开，无内容则保留最小高度

height:auto!important;
height:930px;
min-height:930px;
IE6是不支持min-height属性的。!important 是IE7和FF看 等能够识别的，IE6不识别，并且有绝对优先权。所以 高级浏览器 支持最小高度min-height所以是按min-height的最小高度显示的，而 !important 导致高级浏览器 处理 height:auto，而不处理height:930px了。而IE6则不同，IE6不识别！important而处理height:930px,也不认识min-height。但是 IE6的 height属性本来就是这样的，如果高度每超出height的设定则 高度就是 height高，如果超出 ie6会自动的添加高度。 那么从代码的分层效率上讲建议写个IE6 HACK 减少高级浏览器不必要的处理，在考虑效率而不拘泥于标准的情况下。如下： 
height:auto!important;
_height:930px;
min-height:930px;
