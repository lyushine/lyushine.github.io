第七届 D2前端交流论坛感受

以前在网上看过D2的PPT，也一直希望能够亲身前往前听，这次终于圆了我的想法。可能是由于自己的境界不够吧，感受不多，体会不深，也稍微讲下。这次D2应该是办的最大的一次，不止有主会场，还有各分会场，我听的是D2的主会场第一天。

第一个体会： javascript趋势很强劲，D2主会场讲的都是javascript方面的东西。

第二个体会： 网站的前端性能对于前端开发非常重要，特别是对于访问量大的系统。整个上午都是讲前端性能方面的东西。如何发现前端性能问题、如何优化前端性能、如何监控前端性能。当然我也基本了解了监控前端性能的大概方法，如何优化前端性能。

第三个体会：工具化对公司很重要，整个D2主会场基本都是讲工具，讲演讲者所开发的工具，利用工具解决前端问题，提高工作效率。一个有效的工具能大规模提高团队战斗力。当然，个人觉得写前端的工具是一层境界。

第四个体会：技术分享会基本都是分享新技术或者分享工具，所以在选择听的时候要有自己的取舍，譬如有很多人后来去看分会场的新技术分享了，而我却一直流在原地。应该学会选择取舍。

整个会场让我感受最深的是百度的berg分享的《百度前端集成解决方案》虽然也是讲工具，不过却讲的很不错。讲述如何解决前端开发过程中的历史遗留问题，譬如样式越来越多，旧的样式不敢删 等引起我们的共鸣，并介绍了他的项目将如何全自动解决。

哎，是不是自己境界不够哦，体会不多~~收获的东西也不多。

2012-07-15

大学四年总结

其实已经毕业很多天了，一直各种原因导致我没有对此做一个简单的自我总结。毕业后才知道其实大学生活是多么的美好。岁月毫不留情，转瞬即逝。大学的学习和生活也是我人生中的一大转折点，我很享受这四年的美好生活，也很留恋，但是最终我还是必须不断的前进继续奋斗迎接接下来的一年又一年。

四年的大学生活是总结不完的，不过我我简单的自我总结一下。

学习方面:

从最初的认认真真上课听老师讲课学习，到后来的基本不听课，按自我方向自行学习。实践证明，我的选择是正确的。对以后的工作方面重要的是能有自己拿的出手的一门技能。正如我们专业以前有位说的“其实无所谓学校好不好,专业垃圾不垃圾,前途是光明的,道路是曲折的,老师是靠不住的,一切都是要自学的”。所以大学四年，其主要精力用在了学习CSS上，并且适当的学习了一点photoshop、javascript和php。而迫于学院气氛，尽力把英语四级过了，也拿了一大把荣誉证书，但是后来发现签合同的时候也根本不看这些，全看能力，暂时没有用。实践也证明很多每天上课认认真真听老师讲课的同学找工作的时候不突出，最后大四去报名参加培训然后再找工作。总之，能力代表未来。

工作方面：

在学生会打过酱油，就挂挂职位，锻炼效果不大。不过利用自己学习的css和php(织梦CMS)在整个大学期间也零零散散赚取了万元左右生活费，相对还是比较不错的。在大二期间旷了几个月课在市里一家小网络公司兼职了3个月，期间学院对我各种警告不过后来时间证明都是吓人的，虽然一个月就几百块钱的兼职费，但是在这里我的技术水平在实际的公司项目开发中得到了较大进步。抓住每一次工作锻炼的机会，让自己进步，因为我知道每一次机会来之不易，抓紧眼前的机会。也是因为知道每次机会来之不易，所以那时每天花1-2个小时帮一位学长发帖担任网站编辑（寝室断网了的时候跑网吧发），只是为了跟他学习CSS。大学的四年工作方面挂过职位、做过兼职、赚过私单、摆过地摊、搞过自己的网站。

生活方面：

收获了一些知心朋友，正如有人说虽然我不是那种有很多朋友满天下的那种，但是我对朋友非常用心，所以和我做朋友的基本知心朋友。除了偶尔起早贪黑为了完成项目，我的生活作息还是非常标准的。因为有个时候我喜欢一气呵成，不喜欢拖拉，所以显得激愤而执着造成睡眠晚，不过只是偶尔的特定需求。而生活方面一大不足是平常没有注重锻炼只是偶尔打打球，迫于程序员可持续发展的个人看法我觉得以后应该保持好身体。一大遗憾还有就是自己太宅了，没有利用这四年的时间多玩玩，看看周围的风景。

几点重要的体会：

抓住每一次机会

抓住学长招编辑的机会，我CSS入门了。抓住兼职的机会，我进一步提高了。在校园招聘末期，抓住了最后一次机会，我进入了现在的公司。

学习靠自己

引用室友的信奉名言“古之成大事者不惟有超世之才已有坚韧不拔之志”，现在想想，自己学习的进步主要还是自己一行一行代码敲下来，一个视频一个视频的观看来的。

专注

专注做好自己的事，把一件事情做到极致，方能有所达成。

注重基础

工作的体会，技巧是在基础上得来的，很多没见过的东西是自己用基础“想”出来的。基础方能以不变应万变，花架子是靠不住的。

不断重构自己

人容易犯经验主义错误，人容易满足于目前的状况， 要不断的充实自己、重新认知自己，去除自己不好的东西，吸收好的先进的东西才能进步。切莫固步自封。

目标

大学的这四年相对于以前的人生路上是比较成功的。遗憾的事情就不多说（人往前走）还有很多地方我做的不好，疼痛是成长所必须的。我认为下一个阶段是工作的第一个三年，第一个三年必将是充满期待和自我挑战的三年。路漫漫其修远兮吾将上下而求索，往大牛之路前进吧！

未知宽度水平居中的几种方法

在某些特定的场合，在没有知道宽度的情况下却要求水平居中，前段时间在写分页的时候碰到了这个问题。当时在网上找了一些解决方法，并选了一个最合适的方法，现将当时整理的一些方法发出。下列以分页作为演示说明：

方案一、利用浮动的包裹性和相对定位百分比数据值特性,传说称之为“相对浮动”

.unknow_width_center1 {position:relative;    left:50%;    float:left;}
.unknow_width_center1 li {position:relative;    right:50%; z-index:2;    float:left}</pre>
<ul class=“unknow_width_center1”>
    <li><a href=“#”>1</a></li>
    <li><a href=“#”>2</a></li>
    <li><a href=“#”>3</a></li>
    <li><a href=“#”>4</a></li>
    <li><a href=“#”>5</a></li>
    <li><a href=“#”>7</a></li>
    <li><a href=“#”>8</a></li>
    <li><a href=“#”>9</a></li>
    <li><a href=“#”>10</a></li>
</ul>

通过给父元素设置浮动float,再设置父元素的position属性为relative和left:50%;子元素设置position:relative和left:-50%来实现水平居中。

可以保留块级元素仍然是以display:block的形式显示,不会添加无意义的标签,不添加嵌套深度。缺点是设置了position:relative;带来了一些副作用，并且需要闭合（清除）浮动。

方案二、利用text-align属性特性（text-align:center 对于ie6、7块级和内联级都可以水平居中。其它浏览器内联级(内联块级，文字)元素才可以水平居中，块级无效。 ）强制定义为内联 display:inline

.unknow_width_center2 {text-align:center; padding:5px;}
.unknow_width_center2 li {display:inline;}</pre>
<ul>
    <li><a href=“#”>1</a></li>
    <li><a href=“#”>2</a></li>
    <li><a href=“#”>3</a></li>
    <li><a href=“#”>4</a></li>
    <li><a href=“#”>5</a></li>
    <li><a href=“#”>7</a></li>
    <li><a href=“#”>8</a></li>
    <li><a href=“#”>9</a></li>
    <li><a href=“#”>10</a></li>
</ul>

将子元素设置为内联居中，那么大家都知道的display:inline内联元素宽高是不可以设定的，局限性太大，不宜使用。

方案三、利用table的属性特性

.unknow_width_center3{margin:0 auto}
.unknow_width_center3 ul {overflow:hidden; *zoom:1}
.unknow_width_center3 ul li {float:left;}</pre>
<table class=“unknow_width_center3”>
<tbody>
<tr>
<td>
<ul>
    <li><a href=“#”>1</a></li>
    <li><a href=“#”>2</a></li>
    <li><a href=“#”>3</a></li>
    <li><a href=“#”>4</a></li>
    <li><a href=“#”>5</a></li>
    <li><a href=“#”>7</a></li>
    <li><a href=“#”>8</a></li>
    <li><a href=“#”>9</a></li>
    <li><a href=“#”>10</a></li>
</ul>
</td>
</tr>
</tbody>
</table>

是使用table作为容器的方法来实现。添加了无意义的标签。Table标签本身并不是块级元素,当我们不设置table的宽度的话,他里面的宽度是由他内部元素的宽度撑起来的。但即使我们没有设置table的宽度,直接设置table的外边距margin:0 auto;就可以实现水平居中了!这样我们就可以通过设置table水平居中,间接使里面的内容居中。

方案四、利用内联块display:inline-block的text-align属性

.unknow_width_center4 {text-align:center}
.unknow_width_center4 ul li {display:inline-block; display:inline; zoom:1;}</pre>
<ul class=“unknow_width_center4 “>
    <li><a href=“#”>1</a></li>

未知宽度水平居中的几种方法

在某些特定的场合，在没有知道宽度的情况下却要求水平居中，前段时间在写分页的时候碰到了这个问题。当时在网上找了一些解决方法，并选了一个最合适的方法，现将当时整理的一些方法发出。下列以分页作为演示说明：

方案一、利用浮动的包裹性和相对定位百分比数据值特性,传说称之为“相对浮动”

.unknow_width_center1 {position:relative;    left:50%;    float:left;}
.unknow_width_center1 li {position:relative;    right:50%; z-index:2;    float:left}</pre>

    <li><a href=“#”>1</a></li>
    <li><a href=“#”>2</a></li>
    <li><a href=“#”>3</a></li>
    <li><a href=“#”>4</a></li>
    <li><a href=“#”>5</a></li>
    <li><a href=“#”>7</a></li>
    <li><a href=“#”>8</a></li>
    <li><a href=“#”>9</a></li>
    <li><a href=“#”>10</a></li>
</ul>

通过给父元素设置浮动float,再设置父元素的position属性为relative和left:50%;子元素设置position:relative和left:-50%来实现水平居中。

可以保留块级元素仍然是以display:block的形式显示,不会添加无意义的标签,不添加嵌套深度。缺点是设置了position:relative;带来了一些副作用，并且需要闭合（清除）浮动。

方案二、利用text-align属性特性（text-align:center 对于ie6、7块级和内联级都可以水平居中。其它浏览器内联级(内联块级，文字)元素才可以水平居中，块级无效。 ）强制定义为内联 display:inline

.unknow_width_center2 {text-align:center; padding:5px;}

将子元素设置为内联居中，那么大家都知道的display:inline内联元素宽高是不可以设定的，局限性太大，不宜使用。

方案三、利用table的属性特性

CSS七种方法实现分割中线

在项目开发过程中的会员中心页碰到了一个中线，然后发散思维，要求中间的线自适应高度，想了七种方案。

如图效果：

 

方案一：左边标签设置左边框，右边标签设置左边框，利用负margin边框重叠

.middle_line_wrap1 .row {
    padding-left:100px;    /* 保证dd的位置，不管有没有dt，dd的位置都距离左边这么远 */
    zoom:1         /*闭合浮动，针对IE*/
}
.middle_line_wrap1 .row:after {    /*闭合浮动针对支持该伪类的现代浏览器*/
    content:"\20";
    clear:both;
    visibility:hidden;
    display:block;
    height:0
}
.middle_line_wrap1 dt {
    width:99px;
    margin-left:-100px; /* 保证dd的位置，不管有没有dt，dd的位置都距离左边这么远 */
    float:left;        /*让元素可以定宽，而又保持布局，不可以用inline-block,inline-block会引起文字元素间的莫名间隙*/
    text-align:right;
    position:relative;
    z-index:1;        /*兼容IE6，为什么需要使用，原因不详*/
    border-right:1px solid #CECECE
}
.middle_line_wrap1 dd {
    border-left:1px solid #CECECE;
    float:left;       /*解决dd元素边框不识别多行的总高，若没有，则本元素边框为一行元素的高而不是多行*/
    padding-left:10px;
    margin-left:-1px;
    position:relative;
    z-index:1px;    /*兼容IE6，为什么需要使用，原因不详*/
}
<div class="middle_line_wrap1"><dl class="row"><dt>姓名：</dt><dd>LYuShine</dd></dl></div>

很“保险”（使程序输出的时候左边和右边任意一边内容为空甚至标签不输出都正常显示），多了一个多余标签。

 

方案二：设置父容器左边框，父容器margin-left,子元素父margin拉回。

    border-left:1px solid #CECECE;
    margin-left:100px; /*其实这个使用在.member_info2也是可以的，这里考虑.member_info2为大体盒子，为了不造成不必要的布局影响麻烦*/
}
.middle_line_wrap2 .row {
    padding-left:100px;    /* 保证dd的位置，不管有没有dt，dd的位置都距离左边这么远 */
    zoom:1         /*闭合浮动，针对IE*/
}
.middle_line_wrap2 .row:after {    /*闭合浮动针对支持该伪类的现代浏览器*/
    content:"\20";
    clear:both;
    visibility:hidden;
    display:block;
    height:0
}
.middle_line_wrap2 dt {
    width:100px;
    float:left;
    text-align:right;
    margin-left:-200px; /* 保证dd的位置，不管有没有dt，dd的位置都距离左边这么远 */

    float:left;
    margin-left:-100px;
    padding-left: 10px;
